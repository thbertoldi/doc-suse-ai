[#suse-ai-architecture-building-blocks]
== {productname} building blocks

Linux operating system::
The underlying operating system with the optional {nvidia} GPU driver installed.
We prefer https://documentation.suse.com/sles[{sls}].
If you require an immutable operating system, https://documentation.suse.com/sle-micro[{slema}] is the recommended alternative. 

{kube} cluster::
{kube} cluster managed by {ranchermanager}, ensuring container and application lifecycle management.
We recommend using the {rke2} (https://documentation.suse.com/cloudnative/rke2/) distribution. 

{nvoperator}::
Uses the {nvidia} GPU's computing power and capabilities for processing AI-related tasks. 

link:https://www.suse.com/solutions/security/[{ssecurity}]::
For security and compliance. 

link:https://www.suse.com/solutions/observability/[{sobservability}]::
Provides advanced performance and data monitoring. 

link:https://www.suse.com/products/rancher/storage/[{sstorage}]::
Enterprise-grade storage solution. 

link:https://www.suse.com/products/rancher/virtualization/[{svirtualization}]::
For virtualized workloads. 

link:https://www.suse.com/products/multi-linux-manager/[{smlm}]::
For managing multiple Linux distributions. 

link:https://www.suse.com/products/rancher/application-collection/[{sappco}]::
Provides curated, trusted, compliant and up-to-date applications for {kube}.
Learn more on its dedicated link:https://apps.rancher.io/[Web site] and in the link:https://www.suse.com/products/rancher/application-collection/[product summary].

[#suse-ai-architecture-ai-library]
== {ailibrary} workloads
Following is a list of AI applications that you can find in the {sappco}.
For a complete and up-to-date list, refer to https://apps.rancher.io/stacks/suse-ai. 

link:https://cert-manager.io/[{certmanager}]::
An extensible X.509 certificate controller for {kube} workloads. 

link:https://opensearch.org/[{opensearch}]::
A search and analytics suite for analyzing and visualizing search data. 

link:https://milvus.io/[{milvus}]::
A vector database built for generative AI applications with minimal performance loss. 

link:https://ollama.com/[{ollama}]::
A platform that simplifies the installation and management of large language models (LLM) on local devices. 

link:https://openwebui.com/[{owui}]::
An extensible Web user interface for the {ollama} LLM runner. 

link:https://github.com/vllm-project/vllm[{vllm}]::
A high-performance inference and serving engine for large language models (LLMs). 

link:https://github.com/open-webui/mcpo[{mcpo}]::
The {mcp}-to-OpenAPI proxy server provided by {owui}. 

link:https://pytorch.org/[{pytorch}]::
An open source machine learning framework. 

link:https://mlflow.org/[{mlflow}]::
An open source platform to manage the machine learning lifecycle, including experimentation, reproducibility, deployment and a central model registry. 

link:https://qdrant.tech/[{qdrant}]::
A vector database and similarity search engine for storing, searching and managing high-dimensional vectors.

link:https://docs.litellm.ai/docs/[{litellm}]::
An open source LLM proxy and abstraction layer that lets you interact with many large language model providers through a single, OpenAI-compatible API.
